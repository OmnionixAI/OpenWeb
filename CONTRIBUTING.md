# Contributing to OpenWeb

## Development workflow

1. Install dependencies with `npm install`.
2. Run `npm run lint`, `npm run build`, and `npm test` before opening a pull request.
3. Keep changes focused and document user-facing behavior in the README when appropriate.

## Standards

- TypeScript only for source changes
- Professional terminal output
- Backward-compatible CLI behavior unless a breaking change is intentional and documented
- Apache 2.0 compatible contributions

## Pull requests

- Explain the user-facing problem being solved
- Include verification notes
- Prefer small, reviewable commits
